# Cat Pinterest

Тестовое задание для **ВК**: веб-приложение в духе ленты изображений — лента котиков с [The Cat API](https://thecatapi.com/), избранное и бесконечная подгрузка.

## Возможности

- главная страница с сеткой карточек и подгрузкой по скроллу (infinite scroll);
- добавление котиков в избранное (Zustand), отдельная страница избранного;
- кэширование ленты при переходах между страницами (данные не теряются при уходе с главной);
- деплой на **GitHub Pages** (см. `vite.config.ts` и `.github/workflows`).

## Запуск

```bash
npm install
npm run dev
```

Сборка и превью продакшена:

```bash
npm run build
npm run preview
```

Линт:

```bash
npm run lint
```

## Стек и обоснование выбора

| Технология | Зачем |
|------------|--------|
| **React** | Стандарт для UI с компонентной моделью и экосистемой; удобно описывать карточки, сетку и навигацию. |
| **TypeScript** | Статическая типизация для пропсов, ответов API и стора — меньше ошибок при рефакторинге. |
| **Vite** | Быстрый дев-сервер и сборка, минимальная конфигурация, нативный ESM; удобно для SPA. |
| **React Router** | Клиентский роутинг (`/`, `/favorite`) без перезагрузки страницы, `<Link>` / `NavLink` для навигации. |
| **TanStack Query** | Запросы к API, кэш и **повторное использование данных** при возврате на главную; **`useInfiniteQuery`** для постраничной подгрузки без ручного `useEffect` + `page`. Состояние сервера отделено от UI. |
| **Zustand** | Лёгкий глобальный стор для избранного без лишнего бойлерплейта; доступ по `id` через `Record` — быстрые проверки «в избранном» на каждой карточке. |
| **Sass (SCSS) + CSS Modules** | Локальные классы без коллизий имён, вложенность и переменные там, где удобно для карточки и сетки. |
| **ESLint** | Базовый контроль качества кода и правил React Hooks. |

## API

Используется **The Cat API** (`/v1/images/search`). Для публичного репозитория ключ лучше хранить в переменных окружения (например `VITE_…`) и не коммитить в исходники.
